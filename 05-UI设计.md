# 05 · UI 设计规格 — molesignal-website

> 产研流水线第五棒（UI Designer）。
> 基于 PRD（03）+ UX（04）+ 现有 globals.css/组件体系，给出完整改版视觉语言。
> 目标：不是微调，而是确立一套**有辨识度的新视觉方向**，并落成前端可直接用的 token 规格。
> 编写时间：2026-06-01。

---

## 1. 视觉方向

### 1.1 改版动因——现状诊断

现有设计已有不错的骨架（OKLCH 分层 token、清晰排版阶梯、暗色主题支持），但存在一个核心问题：**它看起来像一个漂亮的通用 SaaS 模板，而非一个硬核工程工具**。

- 品牌色 Indigo（`#3742c7`）+ 营销粉渐变（Indigo→Pink）是高频 AI/SaaS 审美，辨识度趋零。
- 背景色偏"淡蓝冷白"（`--bg-0: #fbfbfd`），整体调性更接近协作 SaaS（Notion / Linear 附近），而非可观测性工具的数据密度质感。
- Hero 区块主要靠居中大字 + 渐变 CTA 驱动——这正是用户已经审美疲劳的模板。
- 等宽字体完全缺席（`--font-mono` 指向 Inter Variable，不是真等宽），这在代码/指标展示密集的工程产品上是硬伤。
- 阴影系统偏"通用卡片感"，缺乏数据工具的精密感。

### 1.2 改版目标关键词

**Raw / Signal / Tight / Proof / Monospace**

| 词 | 含义 |
|---|---|
| **Raw** | 去除装饰层，数据优先。像 `htop`/`psql` 的 CLI 直接感，不过度包装。 |
| **Signal** | 信噪比极高。重要信息突出，次要信息退场，没有空洞的"装饰性文案"。 |
| **Tight** | 间距紧凑但不拥挤，文字密度契合工程师读内容的习惯，不是大图大字小文字的营销节奏。 |
| **Proof** | 数字、命令、diff——一切声明都有证据伴随。视觉上数字/代码是一等公民。 |
| **Monospace** | 等宽字体贯穿数字展示/代码区/指标值，给页面植入"这是个真工具"的质感。 |

### 1.3 情绪基调

不是 Datadog 的企业蓝（权威但冷），不是 Grafana 的橙色仪表盘（可视化为主），也不是通用 SaaS 的紫/粉渐变（精致但无辨识度）。

**参照气质**：
- **Litestream** 官网：深色背景、极简排版、代码块占核心地位、不卖弄设计感
- **Fly.io** 早期官网：工程师直球、文字驱动、CLI 截图而非 Figma 样机
- **PostgreSQL** 生态工具（pganalyze、Tembo）：数字密度高、monospace 指标、深色专业感

**不参照**：Linear（过于精致）、Vercel（黑白极简但没有数据感）、Datadog（过于企业）、通用 AI SaaS 落地页（大渐变 Hero + 三个"功能卡片"）。

### 1.4 核心改版——改了什么、为什么更对

| 维度 | 现状 | 改版 |
|---|---|---|
| **主品牌色** | Indigo `#3742c7`（SaaS 蓝紫，高频） | **Teal `#0d9488`**（实用工具的"可靠绿蓝"，联想终端/命令行，与红 Datadog 形成对比） |
| **营销强调色** | Pink-Magenta（`#ba3071`）渐变 | **Amber `#d97706`**（成本/省钱的"金钱色"，CostCalculator 的数字用它才有冲击力） |
| **背景** | 淡蓝冷白 `#fbfbfd` / 深蓝黑 `#0d101a` | **亮色：暖中性白**（`#f9f9f8`，消除冷蓝），**暗色：深炭黑**（`#0e1117`，更暗、更专业，接近终端） |
| **等宽字体** | Inter Variable（非等宽） | 引入 **Geist Mono**（`@fontsource/geist-mono`，已在 npm），数字/代码用真等宽 |
| **阴影系统** | 常规卡片阴影（柔和弥散） | **边框主导 + 微阴影**：用 1px 实线边框代替阴影做层次，阴影仅用于浮层/聚焦态，更精密 |
| **Hero 处理** | 居中大字 + 渐变文字 + CTA | **左对齐排版 + 终端仿真前缀**（`$`/`>`），Hero 下方放一行真实命令而非空洞副标题 |
| **数字展示** | 普通 font + 高亮色 | **Geist Mono + Amber**，Savings 数字大字号 + tabular nums，像一个真实的监控仪表 |
| **CardBorder** | 渐变/阴影装饰 | 1px `--bd-0` + 左侧 3px Teal 强调线（适用于功能对比卡/changelog 条目），工程文档感 |

## 2. 设计系统

### 2.1 色彩

#### 主色：Teal（功能主色）

Teal 是工程工具的可信色——联想终端绿、数据库客户端、监控曲线。与 Datadog 的蓝企业感形成视觉区隔，更接近"自托管开源工具"的诚实调性。

```
亮色模式：
--brand:       oklch(52% 0.13 172)    /* ≈ #0d9488 teal */
--brand-soft:  oklch(62% 0.13 172)    /* ≈ #14b8a6 */
--brand-dim:   oklch(52% 0.13 172 / 0.12)

暗色模式：
--brand:       oklch(68% 0.13 172)    /* ≈ #2dd4bf — 亮色在暗背景上 */
--brand-soft:  oklch(76% 0.12 172)    /* ≈ #5eead4 */
--brand-dim:   oklch(68% 0.13 172 / 0.16)
```

对比度（亮色模式，白底上）：Teal `#0d9488` 对比白 = **4.6:1**，满足 AA。
对比度（暗色模式，`#0e1117` 底）：亮化 Teal `#2dd4bf` 对比 = **8.1:1**，满足 AAA。

#### 强调色：Amber（省钱/成本信号）

Amber 专用于成本相关区块：CostCalculator 的 Savings 数字、"省 XX%" 徽章、Pricing 对比表的节省高亮。不作为通用品牌色，保持语义唯一性。

```
亮色：
--amber:       oklch(62% 0.17 60)     /* ≈ #d97706 amber-600 */
--amber-soft:  oklch(72% 0.15 60)     /* ≈ #f59e0b */
--amber-dim:   oklch(62% 0.17 60 / 0.12)

暗色：
--amber:       oklch(76% 0.15 70)     /* ≈ #fbbf24 amber-400 */
--amber-soft:  oklch(84% 0.14 70)     /* ≈ #fde68a */
--amber-dim:   oklch(76% 0.15 70 / 0.18)
```

对比度：亮色 Amber `#d97706` 在白底 = 3.2:1（仅作为大号展示用色，不用于正文）。暗色 `#fbbf24` 在 `#0e1117` = 9.4:1。

#### 语义色

```
成功（成功态、✓）：
  亮色: --green: oklch(48% 0.14 152)   /* ≈ #157f3f */
  暗色: --green: oklch(68% 0.14 152)   /* ≈ #3dc26e */

警告（诚实标注 coming-soon / pre-1.0）：
  亮色: --warn:  oklch(60% 0.15 60)    /* ≈ #ca8a04 amber-600 变体，但偏黄 */
  暗色: --warn:  oklch(78% 0.14 72)    /* ≈ #fde047 */

错误（表单校验、限流提示）：
  亮色: --red:   oklch(48% 0.20 25)    /* ≈ #d1252a */
  暗色: --red:   oklch(68% 0.20 25)    /* ≈ #f25b56 */

信息（info badge、pill）：
  亮色: --info:  oklch(48% 0.16 255)   /* ≈ #1f6cbe */
  暗色: --info:  oklch(68% 0.15 255)   /* ≈ #4d97e0 */
```

### 2.2 字体

#### 字族

```css
--font-sans: "Inter Variable", ui-sans-serif, system-ui, sans-serif;
--font-mono: "Geist Mono", ui-monospace, "Cascadia Code", "Fira Code", monospace;
```

**Geist Mono** 是 Vercel 出品的现代等宽字体，设计精良、字形清晰、完整支持 tabular-nums，npm 包 `@fontsource/geist-mono`（约 40KB WOFF2，可按字重子集）。

等宽字体用途（严格）：
- 所有数字展示：成本计算器数字、GitHub Stars 数、统计指标
- 代码块（CodeBlock 组件）
- Changelog 版本号（`v0.x.x`）
- Pill/Badge 中的数字（响应时间、比率）
- 表单输入中的值（可选）

**不能滥用**：正文段落、标题一律用 Inter Variable。Monospace 是信号，滥用则失去语义。

#### 字号/字重阶梯

沿用现有阶梯，做两处调整：

1. **display-3xl 降级使用**：72px 在当前设计中过激，仅用于 `/why` 的 Savings 大数字，其他 Hero 用 `display-2xl`（56px）。
2. **body 基础字号升为 15px**（现为 14px）：阅读密度适中，工程师阅读长文更舒适。

```
/* 正文 */
--text-xs:    11px / 1.4
--text-sm:    12px / 1.45
--text-md:    13px / 1.5
--text-base:  15px / 1.6    ← 从 14px 升为 15px
--text-lg:    17px / 1.55
--text-xl:    19px / 1.45
--text-2xl:   22px / 1.35

/* Display（标题）*/
--text-display-sm:  18px / 1.3  / -0.01em   ← 新增（小标题）
--text-display-md:  24px / 1.25 / -0.02em
--text-display-lg:  32px / 1.2  / -0.025em
--text-display-xl:  44px / 1.1  / -0.03em
--text-display-2xl: 56px / 1.05 / -0.03em
--text-display-3xl: 72px / 1.0  / -0.04em   ← 仅用于 Savings 大数字

/* 等宽（Geist Mono）*/
--text-mono-sm:   12px / 1.4
--text-mono-md:   13px / 1.45
--text-mono-lg:   15px / 1.5
--text-mono-xl:   20px / 1.4    ← 指标卡数字
--text-mono-2xl:  32px / 1.25   ← 统计大数字
--text-mono-3xl:  56px / 1.1    ← Savings 展示数字
```

字重：
```
--font-weight-body:           400   ← 正文降为 400（现为 500，略重）
--font-weight-ui:             500   ← UI 标签、按钮
--font-weight-strong:         600
--font-weight-display:        600
--font-weight-display-strong: 700
--font-weight-mono:           400   ← monospace 正常粗细
--font-weight-mono-strong:    600
```

### 2.3 间距系统

保持现有结构，增加两个小 token：

```
--spacing-1:   4px
--spacing-2:   8px
--spacing-3:   12px
--spacing-4:   16px
--spacing-5:   20px
--spacing-6:   24px
--spacing-8:   32px
--spacing-10:  40px
--spacing-12:  48px
--spacing-16:  64px
--spacing-20:  80px
--spacing-24:  96px

/* 营销区块竖向间距 */
--section-xs:  2rem     ← 新增（内容密集页内分区）
--section-sm:  3rem
--section-md:  5rem
--section-lg:  8rem
--section-xl:  12rem
```

### 2.4 圆角

**整体降一档**——工程工具应偏直角/小圆角，不是"圆润 SaaS"：

```
--radius-xs:   2px    ← 新增（inline badge/tag）
--radius-sm:   3px    ← 从 4px 降为 3px
--radius:      5px    ← 从 6px 降为 5px（默认，输入框/按钮）
--radius-md:   6px    ← 从 6px 保持
--radius-lg:   8px    ← 从 10px 降为 8px
--radius-xl:   12px   ← 新增（大卡片/设计伙伴 CTA 区块）
--radius-full: 9999px ← pill/chip 保持
```

### 2.5 阴影

**新策略：边框主导，阴影极简化**。大多数卡片用 1px 实线 `--bd-0` 表达层次，只在以下场景用阴影：

```
/* 浮层（Dropdown/Sheet/Tooltip） */
--shadow-sm:  0 1px 4px oklch(0% 0 0 / 0.08)
--shadow-md:  0 4px 16px oklch(0% 0 0 / 0.12), 0 0 0 1px oklch(0% 0 0 / 0.04)
--shadow-lg:  0 12px 40px oklch(0% 0 0 / 0.16), 0 0 0 1px oklch(0% 0 0 / 0.06)

/* 聚焦/激活（form input, button active） */
--shadow-focus:  0 0 0 2px var(--bg-0), 0 0 0 4px var(--brand)

/* Glow（主 CTA 按钮悬停，谨慎使用） */
--shadow-glow-brand:  0 0 20px oklch(52% 0.13 172 / 0.30)
--shadow-glow-amber:  0 0 20px oklch(62% 0.17 60  / 0.30)

/* Hero 卡片（CrossSignalDemo 背景卡） */
--shadow-hero-card:   0 0 0 1px var(--bd-0), 0 24px 48px oklch(0% 0 0 / 0.08)

/* 暗色覆盖：阴影更深 */
[data-theme="dark"] --shadow-md:  0 4px 16px oklch(0% 0 0 / 0.50), ...
```

**取消的阴影**：`--shadow-glow-indigo` 和 `--shadow-glow-pink`（原 Indigo/Pink glow，随主色替换一并废弃）。

## 3. 核心组件规格

### 3.1 按钮

#### Primary Button（主 CTA）

```
背景: --brand
文字: white
高度: 38px（md） / 44px（lg，Hero 区）
圆角: --radius（5px）
内边距: px-4（md） / px-5（lg）
字体: 15px font-weight-ui(500) font-sans
图标间距: 6px gap

状态：
  默认:   bg=--brand, text=white
  悬停:   box-shadow=--shadow-glow-brand, bg 亮度 +5%
  按下:   bg 亮度 -8%, scale(0.98), transition 80ms
  禁用:   opacity-40, cursor-not-allowed
  加载中: 显示 spinner（16px），隐藏文字（宽度不变，防止抖动）
```

Tailwind 参考：
```html
<button class="inline-flex h-10 items-center gap-1.5 rounded bg-brand
  px-4 text-[15px] font-medium text-white transition-all
  hover:shadow-glow-brand active:scale-[0.98]
  disabled:opacity-40 disabled:cursor-not-allowed">
```

#### Secondary Button（次 CTA / ghost）

```
背景: transparent
边框: 1px solid --bd-0
文字: --tx-0
高度: 同 Primary

状态：
  默认:   border=--bd-0, text=--tx-0
  悬停:   bg=--bg-hover, border=--bd-1
  按下:   bg=--bg-2
  禁用:   opacity-40

典型用途: Hero 副 CTA "View on GitHub"、表单旁边的 cancel
```

#### Ghost Button（文字链接级按钮）

```
无边框，无背景
文字: --tx-1，悬停: --tx-0
下划线: 悬停时 underline offset-2
用于: "See full comparison →"、"How does this work? →" 等内联导航

禁用态: text=--tx-3, cursor-default（不用 not-allowed）
```

#### Danger Button（表单错误操作，极少用）

```
背景: --red-dim，文字: --red
悬停: bg=--red, text=white
```

### 3.2 输入 / 表单

```
Input:
  高度: 40px
  边框: 1px solid --bd-0，圆角: --radius(5px)
  背景: --bg-0（亮色）/ --bg-2（暗色，略浮起）
  文字: 15px --tx-0，placeholder: --tx-3
  内边距: px-3
  
  聚焦: border-color=--brand, box-shadow=--shadow-focus
  错误: border-color=--red, box-shadow=0 0 0 2px --red-dim
  禁用: bg=--bg-2, text=--tx-2, opacity-60

Label:
  13px font-weight-ui(500) --tx-1
  margin-bottom: 6px

错误文字（行内）:
  12px --red，margin-top: 4px
  使用 aria-describedby 关联 input

Textarea:
  同 Input 但 min-height: 100px, resize: vertical
  字数限制（400字）: 右下角 monospace 小字显示剩余字数
```

### 3.3 卡片

两种卡片层次：

**Surface Card（通用内容卡）**：
```
背景: --surface（= --bg-1）
边框: 1px solid --bd-0
圆角: --radius-lg(8px)
内边距: p-5（默认）/ p-6（宽松）

悬停（可点击卡片）:
  border-color: --bd-1
  background: --bg-hover
  transition: 150ms ease

不用 box-shadow，用边框传达层次。
```

**Accent Card（功能强调卡，如 Design Partner CTA）**：
```
背景: 渐变（见下方 Hero 背景）
边框: 1px solid --brand / 0.25
左侧强调线: 3px solid --brand（可选，单卡片场景）
圆角: --radius-xl(12px)
不使用 shadow-glow，glow 效果已经过度泛滥。
```

**Code / Terminal Card（CrossSignalDemo、QuickStart 代码区）**：
```
背景: --bg-0（亮色）/ --bg-0（暗色，即最深色 #0e1117，强调终端感）
边框: 1px solid --bd-0
头部模拟 terminal bar: 12px 三点（red/amber/green，用真实颜色但不可点）
内边距: px-4 pt-3 pb-4
字体: --font-mono, 13px
行高: 1.7（代码阅读需要）
```

### 3.4 导航 / Footer

**TopNav 改版要点**：

```
高度: 56px（从 64px 降，更紧凑）
背景（未滚动）: transparent
背景（已滚动）: --bg-0 / 0.90 + backdrop-blur(12px)
底部边框（已滚动）: 1px solid --bd-0

Brand 字体: font-mono, 15px, font-weight-mono-strong
  → "molesignal" 用等宽字体，像 CLI 工具名，有辨识度

Active 状态:
  border-bottom: 2px solid --brand（下划线式，非背景填充式）
  text: --tx-0

CTA 按钮: Primary 小号 h-8 px-3 text-sm
```

**Footer 结构**（对应 UX §2.2）：

```
背景: --bg-1（轻微区分）
顶部边框: 1px solid --bd-0
上内边距: 48px（section-sm）

四列均等宽度（桌面）→ 两列（平板）→ 单列（移动）

Font: text-sm font-ui
链接颜色: --tx-2，悬停 --tx-0
禁用态链接（Discord "launching soon"）:
  cursor: default, color: --tx-3
  title/aria-label 说明原因

版权行: text-xs --tx-3，与四列分隔线
```

## 6. 可访问性要求（对前端的约束说明）

1. **所有正文对比度 ≥ 4.5:1**：
   - 亮色：`--tx-0` on `--bg-0` = 约 12:1（远超）
   - 亮色：`--tx-2` on `--bg-0` = 约 4.8:1（刚过 AA，不降 tx-2 的 OKLCH lightness 值）
   - 暗色：`--tx-0` on `--bg-0` = 约 14:1（超）

2. **大显示文字（≥18pt / 24px 粗体或 ≥14pt / 18.67px 粗体）允许 3:1**：
   - Amber 数字（`--amber`）在 `--bg-0` 亮色下 ≈ 3.0:1，仅用于 display-size，满足 AA Large。
   - 暗色模式 Amber `#fbbf24` on `#0e1117` ≈ 9.4:1，全覆盖。

3. **禁用态不能只靠颜色表达**：禁用按钮加 `cursor-not-allowed` + `opacity-40` + `aria-disabled="true"`。

4. **成功/错误状态不能只靠颜色**：绿色成功卡同时有 `✓` 文字符号；红色错误同时有 aria-describedby 行内文字。

5. **等宽字体字号不低于 12px（--text-mono-sm）**：小于此的数字/代码不使用 Geist Mono，回退到 Inter。

6. **减少动效**：Geist Mono 数字滚动动效加 `useReducedMotion()` 守卫（UX 设计文档已要求，此处重申）。

7. **focus-visible**：所有交互元素的焦点环用 `--shadow-focus-token`（2px offset + 4px brand ring），而非浏览器默认蓝色。
